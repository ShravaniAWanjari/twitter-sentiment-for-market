# Quant NLP Research Platform

A comprehensive research environment for benchmarking Transformer-based sentiment models on cryptocurrency markets and integrating them into quantitative trading strategies.

## 🚀 Key Features

### 1. Model Benchmarking & Training
*   **Multi-Model Support**: Benchmarks six distinct architectures: ModernBERT, CryptoBERT, FinBERT, BERT-base, RoBERTa-base, and DeBERTa-v3 (small).
*   **Automated Pipeline**: Unified training and evaluation loop with consistent tokenization and hyperparameter management via `JobManager`.
*   **3-Class Sentiment Schema**: Classifies financial text into **Bearish**, **Neutral**, and **Bullish** segments.
*   **Latency Analysis**: Measures and compares inference speed (ms per tweet) across different model architectures.

### 2. Interactive Research Dashboard
*   **Real-time Monitoring**: Follow training progress and logs through an interactive progress card.
*   **Leaderboard**: Comparative view of models based on Macro F1, Accuracy, and Error Rates.
*   **Artifact Gallery**: Visualizes Confusion Matrices, ROC curves, and attention plots generated during training.

### 3. Backtesting Engine
*   **Sentiment-Gated Strategies**: Combines technical indicators (Momentum, RSI) with NLP sentiment thresholds.
*   **High-Frequency Data**: Grounded in January 2025 BTC tick data for realistic simulation.
*   **Advanced Metrics**: Calculates Sharpe Ratio, Sortino Ratio, Win Rate, Profit Factor, and Drawdown profiles.
*   **Equity Comparisons**: Visualizes the alpha generated by NLP-gated strategies vs. baseline "Buy & Hold" or technical-only strategies.

### 4. Explainable AI (XAI)
*   **Token Attribution**: Uses **Occlusion Sensitivity** to highlight which words drove a specific sentiment prediction.
*   **Stability Metrics**: Evaluates model robustness by measuring score deltas when top-weighted tokens are removed.
*   **Counterfactual Analysis**: Automatically identifies potential text edits (e.g., antonym flips) that would reverse the model's prediction.

### 5. Market GPT & Analysis
*   **Grounded Chat**: An LLM-powered financial assistant (Gemini 1.5/2.0 Flash) that provides research insights strictly based on backtest artifacts.
*   **News Grounding**: Correlates live news headlines (via NewsData.io) with 4-hour forward price impact for "Price-Grounding" validation.
*   **Automated Reporting**: Generates downloadable PDF research reports summarizing model performance and strategy results.

## 🛠 Technical Architecture

### Backend Tech Stack
*   **Framework**: FastAPI with Uvicorn for asynchronous API handling.
*   **Machine Learning**: 
    *   **PyTorch** & **Transformers (Hugging Face)** for model serving and training.
    *   **Accelerate** for optimized training runs.
*   **Data Processing**: **Pandas** and **NumPy** for market data analysis and metric calculation.
*   **Reporting**: **FPDF** for automated PDF generation.
*   **LLM Integration**: **Google GenAI SDK** for grounded research chat.

### Frontend Tech Stack
*   **Framework**: **React** (v18+) with **Vite** for fast builds.
*   **State Management**: React Hooks (`useState`, `useMemo`, `useEffect`).
*   **Styling**: Modern Vanilla CSS with a dark-mode research aesthetic, glassmorphism elements, and responsive layouts.

### Core Components
| Component | Description |
| :--- | :--- |
| `AnalysisEngine` | Handles headline sentiment, XAI logic, news fetching, and grounded chat. |
| `BacktestEngine` | Simulates trading logic, calculates financial metrics, and manages price data. |
| `JobManager` | Orchestrates multi-model training sessions and captures logs/status. |
| `CryptoTransformerBase` | Unified API for different BERT architectures to ensure plug-and-play compatibility. |

### Data Schema
*   **Datasets**: Located in `dataset/`, consisting of labeled CSVs for training and validation.
*   **Results**: `results/` directory stores model weights, error analysis CSVs, and visualization PNGs.
*   **Backtest Artifacts**: `RunArtifact.json` preserves the state of the latest strategy simulation for GPT grounding.

---

## 📁 Directory Structure
*   `backend/`: FastAPI application, engines, and managers.
*   `frontend/`: React components, views, and assets.
*   `experiments/`: Training scripts and model-specific runs.
*   `models/`: Core transformer wrappers and registry.
*   `scripts/`: Utility scripts for data processing and API verification.
*   `results/`: centralized output for all research artifacts.
